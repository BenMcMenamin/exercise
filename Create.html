IMPLEMENTATION:

The first feature that I implemented was making the microbit display a random letter. This was the main feature of my project as if I could not figure it out, I’d have to chance project idea. I used a random integer function to get this work by making the random integers one of 1 or 2. I then made 1 = A and 2 = B by implementing an if/else loop. I then implemented a function that would check to see if the answer is correct or not. This was also coded with an if/else loop. After having the major sections implemented, I started working on the more minor sections. I coded in an LED tick and x that would show depending if you are correct or incorrect. I implemented two different sounds, one will play if the user is correct, the other that will play if the user is incorrect. The final section I implemented was the score, which would be massive when it comes to Thonny and firebase and the graphs. I added this into the check function that if the user is correct, the score should increase by one. I also added a section to the check function that would immediately make the game end once the user was incorrect. During the implementation, the biggest problem I faced was trying to think of a way to tell the user something about their memory. I wanted to tell the user if their memory was improving or decreasing, my only problem was how could I do that. I solved this problem when I thought of the idea to implement the moving average to try and see if their memory was improving or not. To help you understand the code more, here is a detailed explanation of a piece of code I used; This code is part of a function called check, this function checks the answer entered by the user and checks to see whether it is correct or not. I used an if/else loop to complete this function. If the answer given by the user is the same as the question given by the microbit, then the LED’s on the microbit will display a tick. In the else loop, the microbit will show an x if the users answer does not match the microbits question
